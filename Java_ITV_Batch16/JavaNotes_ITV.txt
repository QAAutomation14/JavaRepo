#Naming Convention = Industry Standards follow

1. Project-> File-New-JavaProject
    Do's=== Use Alphabets upper case, Use PascalCase, Use AlphaNumberic     
     e.g.     JavaBasic, SBIPayment, FBModule1  
     
    Don't== Can't use space, special char, starting with numeric value  
            21231Abc, @!@%^^&&*AVhsda36r, 
    
    
2. package->src--rightclick-new-pkg

       domainName.projectName.muduleName.environmentName

       com.google.gmail.qa
       com.google.gmail.dev    
       
    Do's==> Always use lower case, use domain name, 
    com.google.gmail.qa, com.google.gmail.dev, org.java.itv.qa
    
    Don't==> Can't use space, special char, no underscore 
    @!@%^^&&*AVhsda36r.sfaYAFY  34636723.SGAJSGUSAU
    
    
    
3. Class->pkg-rightclick-new-class   Can be noun
    Do's=== Use Alphabets upper case, Use PascalCase, Use AlphaNumberic     
     e.g.     JavaBasic, SBIPayment, FBModule1  
     
    Don't== Can't use space, lower case, special char, starting with numeric value  
            21231Abc, @!@%^^&&*AVhsda36r, 
            
            
 4. Method->inside the class- perform some specific task- verb 
    Do's=== Use Alphabets , Use camelCase, Use AlphaNumeric     
     e.g.     calculateMath(), run(), test() 
     
    Don't== Can't use space, upper case, special char, starting with numeric value  
            21231Abc, @!@%^^&&*AVhsda36r, 
 
  5. variable->inside the class and method-It hold some value. 
    Do's=== Use Alphabets , Use camelCase, Use AlphaNumeric     
     e.g.     calculateMath, fb_run3232, fb_test1 
     
    Don't== Can't use space, upper case, special char, starting with numeric value  
            21231Abc, @!@%^^&&*AVhsda36r,
